# 要件定義: 週目標・日記・KPT 作成時に週や日付を指定できるようにする

## 背景・目的

現状、週目標・日記・KPT の新規作成は「今週 / 今日」に固定されており、
過去の日付や先の週に対してレコードを作成できない。
カレンダーで過去の週を表示したときにレコードを追加したいユースケースや、
入力し忘れた日の日記を後から記録したいニーズに応える。

## 対象ファイル（影響範囲）

設計フェーズで確定する。調査候補:

- `app/goals/new.tsx`: 週目標作成画面 — 週選択UIを追加
- `app/records/daily/new.tsx`: 日記作成画面 — 日付選択UIを追加
- `app/records/kpt/new.tsx`: KPT作成画面 — 週選択UIを追加
- `hooks/use-weekly-goals.ts`: 任意週の作成ロジック
- `hooks/use-daily-records.ts`: 任意日の作成ロジック
- `hooks/use-kpt-records.ts`: 任意週の作成ロジック
- `infrastructure/repositories/WeeklyGoalRepository.ts`: 週指定クエリ
- `infrastructure/repositories/DailyRecordRepository.ts`: 日付指定クエリ
- `infrastructure/repositories/KPTRecordRepository.ts`: 週指定クエリ
- `domain/models/index.ts`: モデル制約の確認
- `services/WeekService.ts`: 任意日付から週の start/end を算出するユーティリティ

## 機能要件

### 週目標（WeeklyGoal）

- [ ] 作成画面に「週を選択」UIを設置する（デフォルト: 今週）
- [ ] 選択した週の `week_start_date` / `week_end_date` でレコードを保存する
- [ ] 同じ週にすでに週目標が存在する場合は作成を禁止し、エラーメッセージを表示する

### 日記（DailyRecord）

- [ ] 作成画面に「日付を選択」UIを設置する（デフォルト: 今日）
- [ ] 選択した日付でレコードを保存する（`created_at` は今日のまま、日付フィールドは `date` カラムで管理）
- [ ] 選択日付に対応する週の週目標と紐づける（`weekly_goal_id`）
- [ ] 同じ日にすでに日記が存在する場合は作成を禁止し、エラーメッセージを表示する
- [ ] 選択した日付に対応する WeeklyGoal が存在しない場合、警告を表示する（保存は可否を設計フェーズで決定）

### KPT（KPTRecord）

- [ ] 作成画面に「週を選択」UIを設置する（デフォルト: 今週）
- [ ] 選択した週の WeeklyGoal と紐づける（`weekly_goal_id`）
- [ ] 同じ週にすでに KPT が存在する場合は作成を禁止し、エラーメッセージを表示する
- [ ] 選択した週に対応する WeeklyGoal が存在しない場合、警告を表示する

## 非機能要件

- オフライン動作: 日付選択・バリデーション・保存はすべてローカル SQLite で完結する
- DDD 規約: 重複チェックロジックはリポジトリ層（または hook 層）に閉じ、UI から直接 DB を参照しない
- 週の開始曜日はユーザー設定（`use-settings.ts`）に従う
- 日付・週の表示フォーマットは既存 UI と統一する
- 週選択 UI は週の月曜日〜日曜日の範囲を表示し、ユーザーが直感的に選べる形にする

## 受け入れ条件

- [ ] 週目標作成画面で今週以外の週を選択して保存できる
- [ ] 同一週に週目標を2件作成しようとするとエラーになる
- [ ] 日記作成画面で今日以外の日付を選択して保存できる
- [ ] 同一日に日記を2件作成しようとするとエラーになる
- [ ] 日記を作成する日付に対応する WeeklyGoal が存在しない場合、ユーザーに通知される
- [ ] KPT 作成画面で今週以外の週を選択して保存できる
- [ ] 同一週に KPT を2件作成しようとするとエラーになる
- [ ] KPT を作成する週に対応する WeeklyGoal が存在しない場合、ユーザーに通知される
- [ ] 週開始曜日設定を変更しても、週の範囲計算が正しく反映される

## 除外スコープ

- 未来の日付への作成制限は今回は設けない（設計フェーズで要検討）
- 既存レコードの日付・週の変更（編集）機能は対象外
- リマインド通知と連動した日付の自動補正は対象外
- プロジェクト（Project）の日付変更は対象外
