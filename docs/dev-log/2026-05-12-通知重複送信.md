# 通知重複送信

**完了日**: 2026-05-12  
**カテゴリ**: バグ修正

## 背景・原因

`useSettings()` フックが初期化時（`useEffect`）に `scheduleAllNotifications()` を呼び出していた。
このフックは `calendar.tsx`・`index.tsx`・`settings.tsx`・`goals/new.tsx` の4画面で個別にインスタンス化されており、アプリ起動時にほぼ同時に複数の `scheduleAllNotifications()` が並走するレースコンディションが発生していた。

`scheduleAllNotifications()` 内部で `cancelAllNotifications()` を呼んでから再スケジュールするが、複数呼び出しが並走すると「キャンセル前にスケジュール済みのものが残る」状態になり、通知が重複登録されていた。

## 対応方針

`useSettings` フックの責務を「設定の読み書き」のみに限定し、起動時の通知スケジュール復元処理をアプリのエントリポイント（`_layout.tsx`）に集約する。

- `useSettings` の `useEffect` から `scheduleAllNotifications(loaded)` 呼び出しを削除
- `_layout.tsx` で `loading` が `true → false` に変わったタイミング（1回のみ）で `scheduleAllNotifications(settings)` を実行する

## 変更ファイル

| ファイル                | 変更内容                                                                                                     |
| ----------------------- | ------------------------------------------------------------------------------------------------------------ |
| `hooks/use-settings.ts` | `useEffect` 内の `scheduleAllNotifications(loaded)` 呼び出しを削除                                           |
| `app/_layout.tsx`       | `useSettings()` を呼び出し、`loading` 完了時に `scheduleAllNotifications()` を1回実行する `useEffect` を追加 |

## 受け入れ条件（確認済み）

- [x] アプリ起動時に通知スケジュール復元が1回だけ実行される
- [x] 複数タブを往来しても通知スケジューリングが発火しない
- [x] 設定変更時は `updateSettings` 経由で引き続き再スケジュールされる
- [x] 静的解析エラーなし

## 備考

`updateSettings` 内の `scheduleAllNotifications(next)` は設定変更時の再スケジュール用として引き続き残している。これは意図した動作（設定変更 → 即時反映）であり、削除対象ではない。
